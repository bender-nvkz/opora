# Политика безопасности

## Поддерживаемые версии

| Версия    | Поддержка безопасности |
|-----------|------------------------|
| main      | ✅ Активная            |
| < 1.0.0   | ❌ Не поддерживается   |

## Сообщение об уязвимостях

**НЕ открывайте публичный GitHub Issue для уязвимостей безопасности.**

### Предпочтительный способ: GitHub Security Advisories

1. Перейдите на https://github.com/opora/opora/security/advisories
2. Нажмите "Report a vulnerability"
3. Заполните форму

### Альтернативный способ: Email

- Описание уязвимости
- Шаги для воспроизведения
- Потенциальное влияние
- Версия/commit в котором обнаружена уязвимость
- Ваши контактные данные (опционально)

**PGP-ключ:** Доступен на keys.openpgp.org (fingerprint публикуется в релизах).

## Время ответа

| Этап | Срок |
|------|------|
| Подтверждение получения | 48 часов |
| Оценка серьёзности | 5 рабочих дней |
| Исправление critical/high | 7 дней |
| Исправление medium/low | 30 дней |

## Процесс

1. Мы подтверждаем получение сообщения
2. Оцениваем серьёзность по CVSS v3.1
3. Разрабатываем исправление в приватной ветке
4. Согласовываем с репортером дату публикации
5. Публикуем исправление + Security Advisory
6. CVE-номер присваивается при необходимости

## Scope

**В scope:** ядро платформы (этот репозиторий), `compose.yaml`, публичное API.

**Вне scope:** сторонние расширения, окружения развёртывания пользователей.

## Признание

Мы благодарим всех кто ответственно сообщает об уязвимостях.
Имена репортеров публикуются в Security Advisory (с их согласия).
